1. Description du projet

Exknot est une plateforme web de type marketplace B2B permettant de mettre en relation des entreprises (Firms) proposant des services professionnels et des clients cherchant à acheter ces services.

Fonctionnalités principales

  • Catalogue de services — Consultation publique avec recherche, filtrage par catégorie et tri par prix
  • Gestion des services — Les entreprises peuvent créer, modifier et supprimer leurs services (CRUD complet avec upload d'images)
  • Panier d'achat — Ajout, modification de quantité et suppression d'articles (stocké en session)
  • Système de commandes — Passage de commande, suivi de statut (pending → validated / cancelled)
  • Avis et évaluations — Notation de 1 à 5 étoiles avec commentaires sur les services
  • Messagerie en temps réel — Chat entre clients et entreprises avec polling AJAX
  • Tableaux de bord — 3 dashboards distincts : Client, Firm, Admin
  • Authentification complète — Inscription, connexion, vérification email, réinitialisation de mot de passe (Laravel Breeze)
  • Gestion de profil — Modification des informations, avatar, bio, pays, nom d'entreprise
  • Pages statiques — Contact, Politique de confidentialité, Conditions d'utilisation
  • Administration — Gestion des utilisateurs (changement de rôle, suppression), statistiques globales

Rôles utilisateur

RôleDescriptionPermissions
clientAcheteur de servicesConsulter, acheter, évaluer, chatter, gérer son profil
firmEntreprise/prestatairePublier des services, gérer les commandes, chatter, dashboard firm
adminAdministrateurAccès total : gestion utilisateurs, statistiques, toutes les opérations

9. Système de rôles et sécurité

Middleware de rôle

Le middleware RoleMiddleware vérifie le rôle de l'utilisateur connecté. Il accepte plusieurs rôles en paramètre :

Route::middleware('role:firm,admin') → Accès réservé aux firms et admins
Route::middleware('role:admin') → Accès réservé aux admins uniquement

Couches de sécurité

MécanismeDescription
Middleware authVérifie que l'utilisateur est connecté
Middleware verifiedVérifie que l'email est confirmé
Middleware roleContrôle d'accès basé sur le rôle (client/firm/admin)
PoliciesVérification de propriété (ownership) sur Product, Order, Review
ValidationValidation serveur de toutes les entrées (FormRequest + validate())
CSRFProtection CSRF automatique de Laravel sur tous les formulaires
HashingMots de passe hashés via Bcrypt (12 rounds)
MustVerifyEmailLe modèle User implémente la vérification d'email
